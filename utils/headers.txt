HTTP Headers Analysis
====================

This file describes the key HTTP request and response headers observed
when interacting with the Open Library API using Postman.


REQUEST 1: Book Search
GET https://openlibrary.org/search.json?title=turing
--------------------------------------------------

REQUEST HEADERS

1. User-Agent: PostmanRuntime/7.51.0
   - Identifies Postman as the client application making the HTTP request.

2. Accept: */*
   - Indicates that the client can accept responses of any media type.

3. Host: openlibrary.org
   - Specifies the target server hosting the Open Library API.

4. Accept-Encoding: gzip, deflate, br
   - Specifies compression formats supported by the client to reduce response size.

5. Connection: keep-alive
   - Requests that the TCP connection remain open for reuse.


RESPONSE HEADERS

1. Content-Type: application/json
   - Indicates that the response body is formatted as JSON.

2. Date: Mon, 12 Jan 2026 06:05:19 GMT
   - The date and time when the server generated the response.

3. Server: nginx
   - Identifies the web server software handling the request.

4. Access-Control-Allow-Origin: *
   - Allows cross-origin requests from any domain.

5. Content-Length: 24583
   - Specifies the size of the response body in bytes.
